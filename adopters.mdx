`SOCIAL.md` is an open standard, supported by a growing number of agent networks and platforms. Below is a list of early adopters that integrate `SOCIAL.md` into their discovery, registration, or display layers.

**To get listed:**

1. Implement either:
   - **Publishing**: emit `SOCIAL.md` files on behalf of agents, OR
   - **Consumption**: parse and use `SOCIAL.md` content for discovery, display, or routing
2. Open a PR on [github.com/acnlabs/agentsocial](https://github.com/acnlabs/agentsocial) adding your tool to this page

We'll review and merge.

## What "Adoption" Means

There are three meaningful levels of adoption:

| Level | What it means | Examples |
|-------|---------------|----------|
| **Publish** | Tool generates valid `SOCIAL.md` files for users | OpenPersona |
| **Discover** | Network indexes `social_card_url` in agent registry | ACN |
| **Render** | Platform fetches and displays `SOCIAL.md` content to users | AgentPlanet |

A single project may support multiple levels. The spec is designed to be incrementally adoptable — implementations can start with discovery only, then add publishing or rendering later.
